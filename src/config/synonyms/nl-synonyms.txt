# Synonym rules in Apache Solr format (one rule per line).
# Blank lines and lines starting with '#' are ignored.
#
# Governance guide:
# - Equivalency, written as a comma list, makes all terms interchangeable in both directions.
#   Use this only for spelling, inflection, abbreviation, or formatting variants with the same intent.
# - Directional rules, written as 'a, b => c', rewrite the left terms to the right canonical term.
#   Use this when the content model deliberately routes a broader or messy term to one known page.
# - Keep clusters strict and small. A term should live in only one cluster.
# - Over-expansion is the #1 precision killer. Each added synonym widens recall and can bury the right page,
#   so review every term before shipping.
# - Test against real query logs before shipping.
# - Treat synonyms as governed content owned by content and business teams, not as ranker internals.
#
# These are generic Dutch civic examples grounded in the bundled sample content.
